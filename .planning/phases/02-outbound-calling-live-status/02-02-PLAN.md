  <name>Task 1: Create Vapi webhook handler and Pusher auth endpoint</name>
  <files>

  <action>
1. Create `src/app/api/webhooks/vapi/route.ts`:
   - POST handler that processes Vapi server messages.
   - Validate the `x-vapi-secret` header against `process.env.VAPI_WEBHOOK_SECRET`. Return 401 if mismatch.
   - Parse JSON body. The body contains `{ message: { type: string, ... } }`.
   - Handle `status-update` message type:
     - Extract `message.status` (one of: "queued", "ringing", "in-progress", "forwarding", "ended")
     - Map Vapi status to app call status:
       - "queued" -> "pending"
       - "ringing" -> "dialing"
       - "in-progress" -> "on_hold" (default for Phase 2; Phase 3 will refine)
       - "ended" -> "completed" (or "failed" if endedReason indicates error)
     - Look up the call record by vapiCallId (from `message.call.id`). If not found, return 200 (idempotent, ignore unknown calls).
     - Insert a `call_event` row:
       - eventType: map Vapi status to call_event_type enum ("dialing" for ringing, "on_hold" for in-progress, "completed" for ended, etc.)
       - metadata: JSON.stringify relevant webhook payload data
     - Update `call` table: set `status` to the mapped status. If status is "dialing" and startedAt is null, set `startedAt = new Date()`. If status is "completed" or "failed", set `endedAt = new Date()`.
     - Trigger Pusher event on `private-call-{callId}` channel:
       ```typescript
       await pusherServer.trigger(callChannel(callId), "status-change", {
         status: mappedStatus,
         timestamp: new Date().toISOString(),
       });
       ```
   - Handle `end-of-call-report` message type:
     - Extract `message.endedReason` and `message.call.id`
     - Look up call by vapiCallId
     - Update call record: `endedReason = message.endedReason`, `status = "completed"` (or "failed" based on endedReason), `endedAt = new Date()`
     - Determine if the call failed based on endedReason. Failed reasons: "customer-busy", "customer-did-not-answer", "twilio-failed-to-connect-call", "assistant-error". Success reasons: "manually-canceled", "exceeded-max-duration", "silence-timed-out" map to "completed".
     - Insert call_event with eventType "completed" or "failed" and metadata containing endedReason
     - Trigger Pusher "call-ended" event with `{ status, endedReason, endedAt }`
   - Handle `hang` message type (Vapi asking if it should hang up):
     - Return `200` with empty JSON (do not hang up -- we want to stay on hold)
   - Always return 200 to Vapi (even on errors -- Vapi retries on non-200)
   - Keep handler lean: one DB lookup, one insert, one update, one Pusher trigger.

2. Create `src/app/api/pusher/auth/route.ts`:
   - POST handler for Pusher private channel authorization.
   - Pusher sends form-encoded body with `socket_id` and `channel_name`.
   - Parse using `request.formData()` to extract socket_id and channel_name.
   - Extract callId from channel_name (strip "private-call-" prefix).
   - Authenticate request using BetterAuth `auth.api.getSession({ headers })`. Return 403 if no session.
   - Verify the user owns this call: query `call` table where `id = callId AND userId = session.user.id`. Return 403 if not found.
   - Authorize using: `const authResponse = pusherServer.authorizeChannel(socketId, channelName);`
   - Return the auth response as JSON.

<task type="auto">
  <name>Task 2: Create call cancel and active call check routes</name>
  <files>

  </files>
  <action>
1. Create `src/app/api/call/cancel/route.ts`:
   - POST handler. Authenticate via BetterAuth. Return 401 if no session.
   - Parse JSON body: `{ callId: string }`. Validate callId is a UUID with zod.
   - Look up call record where `id = callId AND userId = session.user.id`. Return 404 if not found. Return 400 if call is already completed or failed.
   - Cancel the call:
     - If `vapiControlUrl` exists on the call record, POST to it with `{ type: "end-call" }` body.
     - Otherwise, use `vapiClient.calls.delete(call.vapiCallId)` as fallback.
     - Wrap in try/catch -- Vapi may have already ended the call.
   - Update call record: `status = "completed"`, `endedReason = "manually-canceled"`, `endedAt = new Date()`.
   - Insert call_event with eventType "completed" and metadata `{ reason: "user-cancelled" }`.
   - Trigger Pusher "call-ended" event with `{ status: "completed", endedReason: "manually-canceled" }`.
   - Return 200 with `{ success: true }`.

2. Create `src/app/api/call/active/route.ts`:
   - GET handler. Authenticate via BetterAuth. Return 401 if no session.
   - Query call table for the user's most recent call where status is NOT "completed" AND NOT "failed".
   - If found, return 200 with `{ hasActiveCall: true, callId: call.id }`.
   - If not found, return 200 with `{ hasActiveCall: false }`.
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - Both route files exist and export the correct HTTP methods
    - `npm run build` succeeds
  </verify>
  <done>
    Cancel route terminates Vapi call (via controlUrl or SDK delete), updates DB, and triggers Pusher event. Active call route returns the ID of any in-progress call for the authenticated user.
  </done>
