1. Install packages:
   ```bash
   npm install @vapi-ai/server-sdk pusher pusher-js
   ```

2. Update `src/db/schema.ts` -- add two columns to the `call` table:
   - `vapiControlUrl: text("vapi_control_url")` -- nullable, stores Vapi's control URL for hangup/cancel
   - `endedReason: text("ended_reason")` -- nullable, stores Vapi's endedReason value from end-of-call-report

3. Generate and run Drizzle migration:
   ```bash
   npx drizzle-kit generate
   npx drizzle-kit migrate
   ```

4. Create `src/lib/vapi.ts`:
   ```typescript
   import { VapiClient } from "@vapi-ai/server-sdk";
   export const vapiClient = new VapiClient({
     token: process.env.VAPI_API_KEY!,
   });
   ```

5. Create `src/lib/pusher-server.ts`:
   ```typescript
   import Pusher from "pusher";
   export const pusherServer = new Pusher({
     appId: process.env.PUSHER_APP_ID!,
     key: process.env.NEXT_PUBLIC_PUSHER_KEY!,
     secret: process.env.PUSHER_SECRET!,
     cluster: process.env.NEXT_PUBLIC_PUSHER_CLUSTER!,
     useTLS: true,
   });
   ```

6. Create `src/lib/pusher-client.ts`:
   ```typescript
   import PusherClient from "pusher-js";
   let pusherInstance: PusherClient | null = null;
   export function getPusherClient(): PusherClient {
     if (!pusherInstance) {
       pusherInstance = new PusherClient(
         process.env.NEXT_PUBLIC_PUSHER_KEY!,
         {
           cluster: process.env.NEXT_PUBLIC_PUSHER_CLUSTER!,
           channelAuthorization: {
             endpoint: "/api/pusher/auth",
             transport: "ajax",
           },
         }
       );
     }
     return pusherInstance;
   }
   ```

7. Create shared channel name utility `src/lib/call-channel.ts`:
   ```typescript
   export const callChannel = (callId: string) => `private-call-${callId}`;
   ```

   - Authenticate request using BetterAuth `auth.api.getSession({ headers })`. Return 401 if no session.
   - Parse JSON body: `{ ispId: string, issueCategoryId: string, userNote?: string }`
   - Validate with zod: ispId and issueCategoryId are required UUIDs.
   - Check for existing active call by this user (status NOT IN completed, failed). If found, return 409 with `{ error: "You already have an active call", callId: existingCall.id }`.
   - Look up ISP record to get supportPhone. If ISP has no supportPhone, return 400 with `{ error: "ISP has no support phone number configured" }`.
   - Create call record in DB first (status: "pending", userId from session, ispId, issueCategoryId, userNote). This gives us a callId BEFORE calling Vapi.
   - Call Vapi to create outbound call using transient assistant pattern:
     ```typescript
     const vapiCall = await vapiClient.calls.create({
       phoneNumberId: process.env.VAPI_PHONE_NUMBER_ID!,
       customer: { number: ispSupportPhone },
       assistant: {
         name: `EasyCallAI - ${ispName}`,
         model: {
           provider: "openai",
           model: "gpt-4o-mini",
           messages: [{
             role: "system",
             content: `You are an AI assistant calling ${ispName} customer support. Your job is to wait on hold patiently. When asked questions by the automated system, say you need to speak with a representative. Do not provide any personal information. If asked for an account number or details, say "I'll provide that when I speak with a representative." Be polite and patient. The user's issue category is: ${categoryLabel}.`,
           }],
         },
         voice: { provider: "vapi", voiceId: "Elliot" },
         transcriber: { provider: "deepgram", model: "nova-2", language: "en" },
         firstMessage: "Hello",
         firstMessageMode: "assistant-speaks-first",
         silenceTimeoutSeconds: 3600,
         maxDurationSeconds: 2700,
         serverMessages: ["status-update", "end-of-call-report", "hang"],
         server: {
           url: `${process.env.NEXT_PUBLIC_APP_URL}/api/webhooks/vapi`,
           secret: process.env.VAPI_WEBHOOK_SECRET!,
           timeoutSeconds: 20,
         },
       },
     });
     ```
   - Update call record with `vapiCallId: vapiCall.id`. If Vapi provides a `monitor.controlUrl`, store it in `vapiControlUrl`.
   - Insert a call_event with eventType "created".
   - Return 201 with `{ callId: callRecord.id }`.
   - If Vapi call creation fails, update call status to "failed" and return 500 with error.
   - Use try/catch around the entire flow. If DB insert fails, return 500.
