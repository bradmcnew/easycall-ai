       - Export all tables and enums -- BetterAuth and Plan 02/03 will import them

    4. Create `src/data/isps.ts` with static ISP data (source of truth):
       - 6 ISPs: Comcast, AT&T, Spectrum, Verizon, Cox, CenturyLink
       - Each ISP has: `slug`, `name`, `logoUrl` (pointing to `/logos/{slug}.svg`), and `categories` array
       - Categories are PER-ISP (locked decision). Use these realistic categories per ISP:
         - **Comcast**: Billing, Technical Support, Cancellation, General Inquiry
         - **AT&T**: Billing, Technical Support, Cancellation, New Service, General Inquiry
         - **Spectrum**: Billing, Technical Support, Cancellation, Moving Service, General Inquiry
         - **Verizon**: Billing, Technical Support, Cancellation, Plan Changes, General Inquiry
         - **Cox**: Billing, Technical Support, Cancellation, General Inquiry
         - **CenturyLink**: Billing, Technical Support, Cancellation, General Inquiry
       - Each category has: `slug`, `label`, `description` (one-line description for the UI)
       - Export as `ISP_DATA` (typed as const)

    5. Create `src/db/seed.ts`:
       - Import `ISP_DATA` from `@/data/isps`
       - Import `db` from `@/db`
       - Import `isp` and `issueCategory` tables from schema
       - Upsert (insert on conflict do nothing) all 6 ISPs, then all their categories
       - Use `drizzle-orm` conflict handling: `.onConflictDoNothing()` on the slug unique column
       - Make it a runnable script: `tsx src/db/seed.ts` (add tsx as dev dependency if needed)
       - NOTE: The seed script needs to resolve `@/` path aliases. Either:
         (a) Use `tsx --tsconfig tsconfig.json` with paths configured, OR
         (b) Use relative imports in seed.ts only, OR
         (c) Add a `"seed"` script in package.json that uses `tsx` with proper config
       - Recommended: Add `"db:seed": "npx tsx src/db/seed.ts"` to package.json scripts
       - Also add `"db:generate": "npx drizzle-kit generate"` and `"db:migrate": "npx drizzle-kit migrate"` to package.json scripts

    6. Run migrations:
       ```
       npm run db:generate
       npm run db:migrate
       ```

    7. Run seed:
       ```
       npm run db:seed
       ```

    8. NOTE on path aliases in seed.ts: If `@/` imports don't resolve with tsx, switch to relative imports (`../data/isps` instead of `@/data/isps`). The seed script runs outside Next.js, so path aliases may not work without extra config.
